# 바이브코딩 실전 강의안: AI로 업무용 대시보드 만들기

> **강의 목표**: 코딩 경험 없는 수강생 45명이 Claude Code를 활용해 실제 업무용 대시보드를 직접 만들어보는 실습 강의
>
> **환경**: Google Antigravity + Claude Code 에이전트
>
> **강의 시간**: 3시간 (1시간 강의 + 15분 휴식 × 3세션)
>
> **보조강사**: 2명 (개발자) — 15명씩 담당 구역 배정

---

- [ ] Chrome 브라우저 최신버전
- [ ] Google Antigravity 접속 확인 (Claude Code 에이전트)

- [ ] 각 단계별 프롬프트 슬라이드
- [ ] 보조강사 역할 브리핑 (트러블슈팅 체크리스트)

## 완성본 미리보기 (강의 시작 전 5분)

> 강사가 완성된 대시보드를 시연하며 "오늘 3시간 후에 여러분도 이걸 만듭니다"로 동기부여

시연 포인트:
1. 로그인 화면 → 대시보드 진입
2. 탭 전환 (6개 섹션)
3. 차트와 KPI 카드
4. 엑셀 데이터 붙여넣기 → 실시간 반영
5. 챗봇 질의응답

## 1교시: 뼈대 만들기 (50분 강의 + 10분 Q&A)

### 1-1. 바이브코딩이란? (10분 — 슬라이드)

**핵심 메시지**: "코드를 쓰는 게 아니라, 원하는 것을 말로 설명합니다"

```
전통 코딩:  개발자가 코드를 한 줄씩 직접 작성
바이브코딩: AI에게 "이런 거 만들어줘"라고 요청 → AI가 코드 생성
```

**비유**:
- 전통 코딩 = 직접 벽돌 쌓아서 집 짓기
- 바이브코딩 = 건축가에게 "이런 집 지어주세요" 설명하기

**오늘의 도구**: Claude Code (AI 코딩 에이전트)
- 말로 설명하면 파일을 만들고, 수정하고, 실행까지 해줌
- 틀려도 괜찮음 — 다시 설명하면 고쳐줌

### 1-2. 실습 ① 프로젝트 시작하기 (10분)

> **보조강사 역할**: 각 구역 순회하며 Claude Code 접속 확인

**수강생에게 안내**: "지금부터 제가 보여드리는 프롬프트를 그대로 입력하세요"

#### 프롬프트 1: 프로젝트 생성

```
한국어 업무용 대시보드 웹앱을 만들어줘.

조건:
- 프레임워크 없이 순수 HTML, CSS, JavaScript만 사용
- Chart.js CDN으로 차트 구현
- 파일 구조: index.html, css/variables.css, css/layout.css, css/components.css, js/app.js
- 한국어 UI
- 다크 네이비(#1a237e) + 오렌지(#ff8f00) 컬러 테마
- 반응형 디자인
```

> **강사 팁**: 화면에 프롬프트를 띄워놓고, 수강생이 따라 입력할 시간 2분 제공

**예상 결과**: 5개 파일이 생성됨. 수강생에게 "파일이 5개 만들어졌는지 확인해주세요" 안내

### 1-3. 실습 ② 로그인 페이지 만들기 (15분)

#### 프롬프트 2: 로그인 화면

```
login.html 파일을 만들어줘.

요구사항:
- 아이디/비밀번호 입력 폼
- 비밀번호 눈 아이콘 토글 (보기/숨기기)
- 기본 계정: dongkuk1 / 1234
- 로그인 성공하면 index.html로 이동
- sessionStorage에 로그인 상태 저장
- "빠른 로그인" 버튼 추가 (클릭하면 자동 로그인)
- 배경은 그라데이션, 가운데 로그인 박스
- 모든 텍스트는 한국어
```

> **강사 시연**: 먼저 강사가 프롬프트를 입력하고 결과를 보여준 후, 수강생이 따라함

**확인 포인트** (보조강사 체크):
- [ ] login.html 파일이 생성되었는가?
- [ ] 브라우저에서 열었을 때 로그인 폼이 보이는가?
- [ ] "빠른 로그인" 버튼이 동작하는가?

### 1-4. 실습 ③ 대시보드 헤더와 탭 구조 (15분)

#### 프롬프트 3: 메인 레이아웃

```
index.html에 대시보드 레이아웃을 만들어줘.

구조:
1. 상단 헤더 (sticky)
   - 왼쪽: 로고 텍스트 "원료 조달 대시보드"
   - 오른쪽: 조회연도 드롭다운 (2023~2030), 로그아웃 버튼

2. 탭 네비게이션 (sticky, 헤더 아래 고정)
   - 탭1: 부재료실적 모니터링
   - 탭2: 구매실적
   - 탭3: 공장배분
   - 탭4: 등급별현황
   - 탭5: 공지사항

3. 각 탭 클릭하면 해당 섹션만 보이게 전환
4. 기본 활성 탭: 부재료실적 모니터링
5. CSS는 이미 만들어진 variables.css, layout.css, components.css 사용
```

**수강생 확인**: "탭을 클릭해보세요. 내용이 전환되나요?"

> **트러블슈팅 (보조강사용)**:
> - 탭 전환 안 됨 → JS 파일 로드 순서 확인
> - 스타일 깨짐 → CSS 파일 경로 확인
> - 화면 빈칸 → 브라우저 콘솔(F12) 에러 확인

## 2교시: 데이터와 차트 (50분 강의 + 10분 Q&A)

### 2-1. 실습 ④ KPI 카드 만들기 (15분)

> **강사 설명** (3분): "KPI 카드란? 핵심 지표를 한눈에 보여주는 요약 박스입니다"
> 완성본에서 KPI 카드 4개를 보여주며 설명

#### 프롬프트 4: KPI 카드

```
부재료실적 모니터링 탭에 KPI 카드 4개를 추가해줘.

카드 구성:
1. "연간 계획" — 값: 2,725,000톤, 부제: "2024년 수급계획 합계"
2. "누계 실적" — 값: 2,814,877톤, 부제: "계획 대비 누계 입고량", 강조색: accent
3. "달성률" — 값: 103.30%, 부제: "연간 목표 대비 실적", 강조색: success
4. "누계 입고금액" — 값: 9,785억원, 부제: "2024년 기준", 강조색: warning

디자인:
- 가로 4열 배치 (모바일에서는 2열)
- 각 카드에 그림자 효과
- 값은 큰 글씨, 부제는 작은 회색 글씨
- 강조색은 카드 왼쪽 테두리 색상으로 표현
```

**실습 포인트**: "숫자를 자기 회사 데이터로 바꿔보세요"

### 2-2. 실습 ⑤ 차트 추가하기 (20분)

> **강사 설명** (3분): Chart.js가 뭔지, CDN이 뭔지 간단 설명
> "인터넷에 있는 차트 라이브러리를 가져다 쓰는 겁니다"

#### 프롬프트 5: 월별 추이 차트

```
부재료실적 모니터링 탭에 월별 계획 vs 실적 라인 차트를 추가해줘.

데이터:
- X축: 1월~12월
- 계획(파란선): [220000, 230000, 235000, 225000, 220000, 215000, 225000, 230000, 235000, 230000, 225000, 235000]
- 실적(주황선): [228500, 231200, 242100, 226800, 219500, 223400, 231700, 235600, 238900, 227300, 229800, 280077]

차트 설정:
- Chart.js 사용 (CDN: https://cdn.jsdelivr.net/npm/chart.js@4.4.1)
- 애니메이션 비활성화 (성능)
- 계획선은 파란색 점선, 실적선은 주황색 실선
- Y축에 톤 단위 표시
- 호버하면 수치 툴팁 표시
- 차트 컨테이너를 card 안에 넣기
- 카드 제목: "월별 계획 대비 실적 추이"
```

**수강생 확인**: "차트가 보이나요? 마우스를 올려보세요"

#### 프롬프트 6: 도넛 차트

```
같은 탭에 등급별 구성비 도넛 차트를 추가해줘.

데이터:
- 국고상: 0.76%
- 국고중: 28.54%
- 국고하: 64.70%
- 선반설: 6.00%

차트 설정:
- 도넛 차트 (가운데 빈 원형)
- 색상: 파란계열 4단계 (#1a237e, #283593, #5c6bc0, #9ea7dd)
- 범례는 오른쪽에 표시
- 호버하면 톤 수와 비율 표시
- 카드 제목: "등급별 배분 현황"
- 라인 차트 옆에 2열 배치
```

### 2-3. 실습 ⑥ 테이블 만들기 (15분)

#### 프롬프트 7: 데이터 테이블

```
부재료실적 탭 하단에 월별 실적 테이블을 추가해줘.

테이블 구성:
- 헤더: 월 | 계획 | 실적 | 누계실적 | 달성률

- 달성률은 퍼센트 표시

- 테이블 위에 제목: "월간 수급 실적 상세"
- 카드 안에 넣고, 가로 스크롤 지원

데이터 예시:
1월: 계획 220,000 / 실적 228,500 / 누계 228,500 / 달성률 103.86%
2월: 계획 230,000 / 실적 231,200 / 누계 459,700 / 달성률 99.95%
(나머지는 적절히 채워줘)
```

**실습 포인트**: "헤더를 클릭해서 정렬이 되는지 확인해보세요"

## 3교시: 인터랙션과 완성 (50분 강의 + 10분 Q&A)

### 3-1. 실습 ⑦ 엑셀 데이터 붙여넣기 기능 (15분)

> **강사 설명** (3분): "실제 업무에서는 엑셀 데이터를 대시보드에 반영해야 합니다"

#### 프롬프트 8: 데이터 입력 그리드

```
부재료실적 탭에 엑셀 데이터 붙여넣기 기능을 추가해줘.

요구사항:
- 입력 그리드 테이블 (5열: 날짜, 거래처, 등급, 단가, 금액)
- 엑셀에서 복사한 데이터를 그리드에 붙여넣기(Ctrl+V) 가능
- "데이터 반영" 버튼 클릭하면 차트와 KPI에 즉시 반영
- "초기화" 버튼으로 원래 데이터로 복원
- 붙여넣은 건수를 토스트 메시지로 표시
- 그리드는 접을 수 있는 패널(토글) 안에 배치
- 데이터가 100건 이상이면 페이지네이션 (100건씩)
- localStorage에 저장해서 새로고침해도 유지
```

> **실습**: 강사가 준비한 엑셀 파일에서 데이터 복사 → 붙여넣기 시연
> 수강생도 배포된 엑셀 파일로 직접 실습

### 3-2. 실습 ⑧ 챗봇 추가하기 (15분)

> **강사 설명** (2분): "대시보드에 질의응답 챗봇을 붙여봅시다"

#### 프롬프트 9: 플로팅 챗봇

```
화면 오른쪽 하단에 플로팅 챗봇을 추가해줘.

요구사항:
- 둥근 버튼(FAB) 클릭하면 챗봇 팝업 열림
- 챗봇 아이콘은 🤖 이모지 사용
- 팝업 구성: 상단 헤더, 메시지 영역, 하단 입력창
- 키워드 기반 자동응답:
  - "달성률" → 현재 KPI 달성률 수치 안내
  - "계획" → 연간 계획량 안내
  - "인천" → 인천공장 배분 현황 안내
  - "포항" → 포항공장 배분 현황 안내
  - 그 외 → "해당 질문에 대한 데이터를 찾지 못했습니다"
- 메시지 말풍선 스타일 (사용자: 오른쪽 파란색, 봇: 왼쪽 회색)
- 팝업 바깥 클릭하면 닫힘
- 부드러운 열기/닫기 애니메이션
```

**실습 포인트**: "챗봇에 '달성률'이라고 입력해보세요"

## 부록

### A. 보조강사 트러블슈팅 가이드

| 증상 | 원인 | 해결 |
|------|------|------|
| 화면이 안 뜸 | 파일 경로 오류 | 폴더 구조 확인, index.html 위치 확인 |
| 차트가 안 보임 | Chart.js CDN 로드 실패 | 인터넷 연결 확인, script 태그 확인 |
| 탭 전환 안 됨 | JS 에러 | F12 → Console 탭에서 빨간 에러 확인 |
| 스타일이 깨짐 | CSS 파일 누락 | link 태그 확인, 파일명 오타 확인 |
| 붙여넣기 안 됨 | 이벤트 핸들러 누락 | paste 이벤트 리스너 확인 |
| 데이터 사라짐 | localStorage 초과 | F12 → Application → Storage 확인 |
| Claude가 응답 없음 | API 과부하 | 30초 대기 후 재시도 |
| 프롬프트 결과 다름 | AI 응답 변동성 | "다시 만들어줘" 또는 더 구체적으로 재요청 |

### B. 수강생 자주 묻는 질문

**Q: 프롬프트를 똑같이 입력했는데 결과가 다릅니다**
> A: AI는 매번 조금씩 다른 결과를 줍니다. 정상입니다. 원하는 부분만 "이 부분을 이렇게 수정해줘"라고 추가 요청하세요.

**Q: 에러가 나는데 어떻게 하나요?**
> A: 에러 메시지를 그대로 복사해서 Claude에게 "이런 에러가 나는데 고쳐줘"라고 보내면 됩니다. 이것도 바이브코딩입니다!

**Q: 내 회사 데이터로 바꾸려면요?**
> A: 프롬프트에서 숫자와 항목명만 바꿔서 다시 요청하면 됩니다.

**Q: 이걸 실제로 회사에서 쓸 수 있나요?**
> A: 내부 도구로는 충분합니다. 외부 서비스로 배포하려면 보안, 인증, 서버 등 추가 작업이 필요합니다.

### C. 강의 후 자습 과제 (선택)

1. **색상 테마 바꿔보기**: "전체 색상 테마를 초록+흰색으로 변경해줘"
2. **탭 추가하기**: "사용자관리 탭을 추가하고, 사용자 목록 CRUD 기능을 만들어줘"
3. **차트 추가하기**: "거래처별 월간 입고량 막대 차트를 추가해줘"
4. **엑셀 내보내기**: "테이블 데이터를 엑셀 파일로 다운로드하는 버튼을 만들어줘"
5. **다크모드**: "다크모드 토글 버튼을 헤더에 추가해줘"

### D. 프롬프트 작성 팁 (수강생 배포용)

```
좋은 프롬프트의 3가지 원칙:

1. 구체적으로 — "예쁘게 만들어줘" ❌ → "파란색 배경, 흰색 글씨, 둥근 모서리" ✅
2. 구조적으로 — 번호 매겨서 요구사항 나열
3. 예시와 함께 — "예: 1월 220,000톤, 2월 230,000톤"

수정 요청할 때:
- "전체 다시 만들어줘" ❌ → "3번 카드의 색상만 빨간색으로 바꿔줘" ✅
- 작은 단위로 요청하면 실패 확률이 줄어듭니다
```
