## 1. 시스템 개요

동국제강 원료기획팀의 부재료 구매 실적을 모니터링하는 웹 대시보드입니다. 프레임워크 없이 순수 HTML/CSS/JavaScript로 구성되며, Python 스크립트로 Excel 데이터를 변환하여 정적 파일로 서빙합니다.

**핵심 데이터 흐름:**

```
XLSX 엑셀 파일
    ↓  (build_dashboard_data.py)
js/dashboard-data.js  (window.dashboardData)
    ↓  (브라우저 로드)
app.js  →  DOM 렌더링 + Chart.js 시각화
```

## 2. 기술 스택 요약

### 프론트엔드

| 기술 | 버전 | 용도 |
|------|------|------|
| HTML5 | — | 페이지 구조 (login.html, index.html) |
| CSS3 | — | 디자인 토큰 + 반응형 레이아웃 (4개 파일) |
| Vanilla JavaScript | ES6+ | 프레임워크 없는 순수 JS, IIFE 패턴 |

| Supabase JS | 2.43.4 | 인증 + 클라우드 동기화 (ES Module CDN) |

### 백엔드 / 데이터

| 기술 | 용도 |
|------|------|
| Python 3 (표준 라이브러리만) | XLSX → JS 데이터 변환 |
| Supabase (PostgreSQL) | 사용자 인증 + 교차 기기 데이터 동기화 |
| Vercel Serverless | 환경변수 API 엔드포인트 (api/config.js) |

### 인프라

| 기술 | 용도 |
|------|------|
| Vercel | 정적 호스팅 + 자동 배포 |
| GitHub | 소스 코드 관리, main 브랜치 push 시 자동 배포 |
| jsDelivr / esm.sh | CDN (Chart.js, docx, Supabase) |

## 3. 데이터 생성 파이프라인

**파일:** `scripts/build_dashboard_data.py` (561줄)

### 실행 방법

```bash
python3 scripts/build_dashboard_data.py --source 원본파일.xlsx --output js/dashboard-data.js
```

### 처리 과정

1. **XLSX 파싱** — `.xlsx` 파일을 ZIP으로 열고 내부 XML을 직접 파싱 (openpyxl/pandas 불필요)
   - `xl/sharedStrings.xml` → 문자열 조회 테이블
   - 시트 XML → ElementTree로 셀 데이터 추출
   - Excel 날짜 직렬 번호 → `datetime(1899,12,30) + timedelta(days)` 변환

2. **데이터 집계** — 시트별 원본 데이터를 다음 구조로 변환:
   - 월별 구매 요약 (수량, 금액, 평균 단가)
   - 공급업체별 요약 (총 수량, 금액, 이행률, 월별 추이)
   - 매크로 등급 믹스 (5개 대분류별 비중)
   - 공장 배분 (인천/포항, 월별 내역)

3. **출력** — `window.dashboardData` 전역 변수로 JS 파일 생성

### 출력 데이터 구조

```javascript
window.dashboardData = {
  meta: {
    sourceFile, generatedAt, defaultYear: "2024",
    availableYears: ["2023", "2024"]
  },
  years: {
    "2024": {
      overview:    { annualTarget, cumulativeActual, attainmentRate, ... },
      plan:        { monthly: [...], chart: {labels, plan, actual} },
      suppliers:   { averagePerformance, shareChart, trendChart, table },
      purchases:   { totalQty, totalAmount, avgUnitPrice, monthly },
      allocation:  { incheon: {...}, pohang: {...}, monthly },
      gradeImport: { currentYear, compareYear, mix, compareMix, ... }
    },
    "2023": { ... }
  }
}
```

## 4. 인증 플로우

**파일:** `login.html`

### 인증 경로 (2단계 폴백)

```
사용자 접속 → login.html
    ↓
[Supabase 사용 가능?]
    ├─ YES → Supabase Auth (email+password) → profiles 테이블 검증
    └─ NO  → 로컬 폴백 (localStorage usersData 또는 하드코딩 기본값)
    ↓
sessionStorage.loggedInUser = {id, uid, name, dept, role}
    ↓
index.html로 리다이렉트
```

### Supabase 인증 상세

1. `/api/config` (Vercel 서버리스) 또는 `/config.json` (로컬 폴백)에서 Supabase URL/키 로드
2. `supabase.auth.signInWithPassword({email: username@dk.internal, password})`
3. `profiles` 테이블에서 사용자 프로필 조회 → `status !== "inactive"` 검증
4. 성공 시 UUID를 포함한 세션 정보 저장

### 로컬 폴백 인증

- `usersData` (localStorage) 배열에서 ID/비밀번호 일치 확인
- 기본 테스트 계정: `dongkuk1 / 1234` (관리자), `user1 / 1234` (일반)

### 세션 보호

- `index.html` 상단 인라인 스크립트에서 세션 없으면 `login.html`로 리다이렉트
- `login.html`에서 세션 있으면 `index.html`로 자동 이동

## 5. 앱 초기화 시퀀스

**파일:** `index.html` → 각 JS 파일 순차 로드

### 로드 순서 (순서 중요)

```
1. index.html <head> 인라인 스크립트
   └─ window.__supabaseConfigReady (Promise) 생성
   └─ /api/config → /config.json 2단계 폴백으로 Supabase 설정 로드
   └─ window.SUPABASE_URL, window.SUPABASE_ANON_KEY 전역 설정

2. js/dashboard-data.js
   └─ window.dashboardData 전역 등록 (Python으로 생성된 데이터 번들)

3. js/storage.js
   └─ window.appStorage 초기화 (IndexedDB + localStorage 하이브리드)
   └─ IndexedDB "dongkuk_dashboard" DB 오픈, kv 오브젝트 스토어 생성
   └─ localStorage → IndexedDB 마이그레이션 (최초 1회)

4. js/supabase-storage.js (ES Module)
   └─ Supabase 클라이언트 생성
   └─ window.appStorage를 Supabase 호환 API로 교체
   └─ window.__appStorageReady Promise 생성

5. js/admin-features.js
   └─ 공지사항, 일정, 사용자 CRUD 모달 바인딩

6. js/app.js (IIFE)
   └─ __appStorageReady 대기
   └─ sessionStorage에서 로그인 사용자 추출
   └─ appStorage.setUserId(uuid, appId) → prefetch() 트리거
   └─ runMainApp() 호출 → 활성 탭 렌더링
```

## 6. 스토리지 아키텍처

### 3계층 저장소 구조

```
┌────────────────────────────────┐
│  Supabase PostgreSQL (클라우드)  │  ← 교차 기기 동기화 (선택적)
│  RLS 정책, user_id 기반 격리     │
└──────────────┬─────────────────┘
               ↕ 비동기 동기화
┌────────────────────────────────┐
│  IndexedDB (주 저장소)           │  ← dongkuk_dashboard DB, kv 스토어
│  인메모리 캐시 (_cache) 병행      │
└──────────────┬─────────────────┘
               ↕ 폴백
┌────────────────────────────────┐
│  localStorage (폴백)            │  ← IndexedDB 불가 시 사용
└────────────────────────────────┘

┌────────────────────────────────┐
│  sessionStorage (세션)          │  ← 탭별 로그인 정보만
└────────────────────────────────┘
```

### 영속 데이터 키 (7개)

| 키 | 내용 |
|----|------|
| `planClipboardDataByYear` | 연도별 월간 계획/실적 (4행 × 12개월) |
| `rawTransactionDataByYear` | 연도별 사용자 붙여넣기 거래 데이터 |
| `gradeMacroMappings` | 등급→대분류 매핑 (버전 카운터 포함) |
| `supplierAdminItems` | 공급업체 관리 버퍼 |
| `noticesData` | 공지사항 목록 |
| `schedulesData` | 팀 캘린더 일정 |
| `usersData` | 사용자 계정 목록 |

### storage.js API

```javascript
appStorage.ready          // Promise: DB 초기화 + 마이그레이션 완료
appStorage.getSync(key)   // 동기 인메모리 캐시 읽기
appStorage.set(key, val)  // 캐시 우선 쓰기, 비동기 IDB 저장
```

### Supabase 동기화 (supabase-storage.js)

- **Prefetch:** 로그인 시 6개 테이블 일괄 로드 (1000행 배치 페이지네이션)
  - `notices`, `schedules`, `supplier_admins`, `grade_mappings`, `plan_data`, `transactions`
- **Save:** 캐시 즉시 업데이트 → IndexedDB 비동기 쓰기 → Supabase 비동기 upsert
- **User ID 매핑:** Supabase UUID ↔ 앱 사용자 ID 자동 감지

## 7. 메인 앱 로직

**파일:** `js/app.js` (~4,800줄, IIFE)

### 상태 관리

```javascript
const state = {
  selectedYear: "2024",                    // 활성 연도
  planOverrides: { year → {pastedAt, monthly} },  // 수동 계획 편집
  rawTransactionsByYear: { year → [rows] },        // 붙여넣기 거래 데이터
  gradeMappings: { macro → [detailed grades] },    // 등급 매핑 설정
  supplierAdminItems: [],                          // 공급업체 CRUD 버퍼
  // + UI 상태 (페이지, 정렬, 검색어 등)
}
```

### 2단계 캐시 시스템

| 계층 | 대상 | 키 형식 |
|------|------|---------|
| `_txCache` | `getRawTransactionsForYear()` 결과 | `"${year}:${rowCount}:${_gradeMappingsVersion}"` |
| `_aggCache` | 파생 데이터셋 (suppliers, purchases, gradeImport) | 동일 |

**캐시 무효화 (`_invalidateTxCache()`):**
- 연도 변경 시
- 데이터 초기화 시
- 등급 매핑 변경 시

### 핵심 데이터 함수

| 함수 | 역할 |
|------|------|
| `getRawTransactionsForYear(year)` | 거래 데이터 로드 + 등급 매핑 적용 (인플레이스) |
| `buildSupplierDatasetFromTransactions()` | 공급업체×월 그룹핑 (상위 3개) |
| `buildPurchasesDatasetFromTransactions()` | 월별 수량/금액/단가 집계 |
| `buildGradeImportDatasetFromTransactions()` | 대분류 믹스 + 전년 비교 |
| `_invalidateTxCache()` | 양쪽 캐시 모두 클리어 |

### 성능 최적화

- 핫 패스에서 `for` 루프 사용 (`forEach`/`map` 대신)
- `Float64Array`로 수치 누적 계산
- 20k행 초과 시 `setTimeout(0)` 기반 비동기 청크 파싱

## 9. 데이터 붙여넣기 → 렌더링 흐름

```
1. 사용자가 Excel에서 데이터 복사 → 붙여넣기 이벤트 발생
   ↓
2. 행 수 판단
   ├─ ≤20,000행: parseRawTransactionText() (동기 처리)
   └─ >20,000행: parseRawTransactionTextAsync() (20k행/청크, setTimeout(0))
   ↓
3. state.rawTransactionsByYear[year]에 저장
   ↓
4. savePlanToDB() → appStorage.set() → IndexedDB + Supabase 비동기 저장
   ↓
5. 렌더링 시 getRawTransactionsForYear()
   └─ 등급 매핑 인플레이스 적용
   └─ _txCache에 캐시
   ↓
6. 데이터셋 빌더 호출 (차트/테이블 렌더러에서)
   ├─ buildSupplierDatasetFromTransactions()
   ├─ buildPurchasesDatasetFromTransactions()
   └─ buildGradeImportDatasetFromTransactions()
   ↓
7. Chart.js + DOM 테이블 렌더링
```

## 10. 차트 렌더링 파이프라인

### Chart.js 설정

- **애니메이션 비활성화:** `Chart.defaults.animation = false` (전역)
- **커스텀 플러그인:** `valueLabelPlugin` — 막대/도넛 위에 수치 표시
- **테마 연동:** `getCssVar()`, `getSurfaceColor()`로 다크 모드 대응

### 차트 관리 패턴

```javascript
// 인스턴스 레지스트리
chartInstances[key] = new Chart(ctx, config);

// 재생성 전 기존 인스턴스 파괴 (메모리 누수 방지)
destroyChart(key);

// 데이터 없을 때 빈 메시지 표시
setEmptyChartMessage(canvas, "데이터가 없습니다");
```

### 렌더링 동기화

- 이중 `requestAnimationFrame` 사용 → DOM 레이아웃 완료 후 캔버스 렌더링 보장

## 11. DOCX 보고서 내보내기

**트리거:** 내보내기 버튼 클릭 → `exportDocx()`

### 처리 과정

```
1. 숨겨진 탭 임시 활성화
   └─ 모든 데이터 탭 (plan, purchases, gradeImport) 렌더링
   └─ visibility: hidden, position: absolute (화면에 보이지 않음)

2. 차트 이미지 캡처
   └─ captureAllChartImages()
   └─ chartInstances 순회 → canvas.toDataURL() → base64 PNG

3. DOCX 섹션 구성
   ├─ 표지: 제목, 연도, 생성일, 사용자명
   ├─ buildDocxPlanSection()     — 계획 vs 실적, 공급업체 KPI
   ├─ buildDocxPurchasesSection() — 월별 구매, 등급 믹스
   └─ buildDocxGradeImportSection() — 등급 비교, 수입 선적

4. 패키징 및 다운로드
   └─ new docx.Document({sections})
   └─ docx.Packer.toBlob()
   └─ Blob → a.click() 다운로드
   └─ 파일명: 동국제강_원료기획팀_{year}년_보고서_{today}.docx
```

## 12. 관리자 기능

**파일:** `js/admin-features.js` (~1,200줄)

### 공지사항 관리

- CRUD (생성/조회/수정/삭제)

## 15. 전체 워크플로우 다이어그램

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 로그인                                                       │
│     사용자 → login.html → Supabase Auth | 로컬 폴백               │
│     저장: sessionStorage.loggedInUser                             │
└───────────────────────────┬─────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  2. 초기화                                                       │
│     index.html (Supabase 설정 로드)                               │
│         ↓                                                        │
│     storage.js (IndexedDB 초기화 + 마이그레이션)                    │
│         ↓                                                        │
│     supabase-storage.js (사용자 데이터 prefetch)                   │
│         ↓                                                        │
│     admin-features.js (공지/일정/사용자 로드)                       │
│         ↓                                                        │
│     app.js (runMainApp → 활성 탭 렌더링)                          │
└───────────────────────────┬─────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  3. 데이터 상호작용                                                │
│                                                                   │
│  a) 대시보드 조회                                                  │
│     연도 선택 → renderPlan / renderSuppliers / renderGrade         │
│                                                                   │
│  b) Excel 데이터 붙여넣기                                          │
│     paste 이벤트 → parseRawTransactionText()                      │
│     → fillPlanPasteGrid() → applyPlanPasteInput()                │
│     → savePlanToDB() (IndexedDB + Supabase)                      │
│                                                                   │
│  c) 등급 매핑 관리                                                 │
│     편집 모달 → saveGradeMappings()                               │
│     → 캐시 무효화 → 재렌더링                                       │
│                                                                   │
│  d) 관리자 CRUD (공지, 일정, 사용자)                                │
│     모달 폼 → 생성/수정/삭제 → writeStoredData()                   │
│                                                                   │
│  e) 챗봇 질의                                                     │
│     자연어 입력 → parseVerifiedChatQuery()                        │
│     → buildVerifiedXXXAnswer() → 응답 표시                        │
└───────────────────────────┬─────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  4. 내보내기 및 동기화                                             │
│                                                                   │
│  a) DOCX 보고서 내보내기                                           │
│     내보내기 클릭 → 모든 탭 렌더링(숨김)                             │
│     → 차트 캡처 → DOCX 구성 → Blob 다운로드                        │
│                                                                   │
│  b) 교차 기기 동기화                                               │
│     appStorage.set() → IndexedDB 캐시 + Supabase 비동기 저장       │
│     다음 로그인 → prefetch() → 모든 사용자 데이터 로드               │
│                                                                   │
│  c) 오프라인 폴백                                                  │
│     네트워크 오류 → IndexedDB 캐시 데이터 서빙                       │
└───────────────────────────┬─────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  5. 로그아웃                                                     │
│     로그아웃 클릭 → supabase.auth.signOut()                       │
│     → sessionStorage 제거 → login.html 리다이렉트                  │
└─────────────────────────────────────────────────────────────────┘
```
